# Инструкция для тестировщика — staging p2pcs.ru

Пошаговое руководство для ручного тестирования **R.I.P. Market** на закрытом staging.

**URL:** https://p2pcs.ru  
**Дата актуализации:** июль 2026

---

## 1. Что тестируем на этом staging

| Включено | Описание |
|----------|----------|
| ✅ Вход через **Steam** | Реальная авторизация OpenID |
| ✅ **Реальный инвентарь** CS2 | Предметы подтягиваются из Steam-аккаунта |
| ✅ **Единый аккаунт** buy + sell | Один пользователь может и покупать, и продавать |
| ✅ Выставление лотов | Продажа скинов на маркетплейсе |
| ✅ Покупка / отмена / споры | С mock-завершением сделок |
| ✅ **Тестовое пополнение** | Кошелёк → «Тестовое пополнение» |
| ✅ Mock **Admin** | Завершение сделок, модерация (ops) |
| ✅ **Chrome-расширение** (опционально) | Автоотправка trade offer — см. [сценарий G](#сценарий-g-расширение--автоотправка-trade-offer) |
| ❌ USDT / crypto-кошелёк | Не подключён |
| ⚠️ Завершение сделки | Mock Admin (или расширение + reconcile offerId) |

## 2. Перед началом

### 2.1. Требования

- Браузер: **Chrome** или **Firefox** (последняя версия)
- Аккаунт **Steam** с игрой CS2
- Инвентарь CS2 в Steam: **публичный** (не «только для друзей»)
- Желательно **tradable** скины (медали и значки видны, но их нельзя выставить)

### 2.2. Как открыть сайт

| Правильно | Неправильно |
|-----------|-------------|
| https://p2pcs.ru/login | api.p2pcs.ru |
| https://www.p2pcs.ru/login | http без редиректа на старый кэш |

Если сайт не открывается:

1. **Ctrl+Shift+R** (Mac: **Cmd+Shift+R**) — жёсткое обновление
2. Режим **инкогнито**
3. Отключить VPN
4. Написать разработчику: скрин + F12 → Console

### 2.3. Важно про аккаунты

**Один Steam = один пользователь на платформе.** Этот же аккаунт может **и покупать, и продавать** — отдельные роли Buyer/Seller в интерфейсе больше не нужны.

- Нельзя привязать два разных Steam к одному аккаунту
- Нельзя смотреть инвентарь чужого Steam, будучи залогиненным под другим пользователем
- **Mock Admin** — общий тестовый аккаунт для операций (завершение сделок, модерация)
- **Mock Buyer / Seller** — только для автотестов и отладки; **не** основной путь ручного QA

**Типичная ошибка:** вошли через Steam, потом привязали Steam к mock-аккаунту — Steam «переехал» на другого пользователя. Старая сессия в браузере показывает пустой инвентарь.

**Решение:** **Выйти** → войти снова через **Steam**.

## 3. Роли и способы входа

Страница входа: https://p2pcs.ru/login

### 3.1. Steam (основной способ)

1. Вкладка **Steam**
2. **Sign in with Steam**
3. Подтвердить вход в Steam
4. После входа вы попадёте в **каталог** (`/catalog`)

У каждого нового Steam создаётся **отдельный** пользователь со **своим** инвентарём. Один и тот же пользователь может открыть **Сделки** (`/deals`) и увидеть покупки, продажи и свои лоты.

### 3.2. Mock (только для Admin / ops)

1. Вкладка **Mock**
2. Выбрать роль: **Admin**
3. **Continue as mock Admin**

| Роль | Зачем нужна на staging |
|------|------------------------|
| **Admin** | Завершить сделку (mock trade), споры, модерация |
| **Buyer / Seller** | Только для разработчиков и CI; ручной QA — через Steam |

**Рекомендация для полного цикла сделки:**

- **Окно 1:** Steam-аккаунт A (продавец и/или покупатель)
- **Окно 2:** Steam-аккаунт B (второй участник сделки)
- **Окно 3:** Mock **Admin** (инкогнито) — завершение сделки

## 4. Подготовка аккаунта (buy + sell)

Выполните **один раз** на странице **Аккаунт** (`/account`) перед торговлей.

Вверху страницы — блок **«Как начать торговать»** с чеклистом шагов.

| Шаг | Действие | Ожидание |
|-----|----------|----------|
| 4.1 | Войти через **Steam** | Успешный вход, отображается ник |
| 4.2 | Открыть **Аккаунт** (меню профиля) | Страница `/account`, виден onboarding |
| 4.3 | Шаг 1: **Привязать Steam** | Steam ID отображается; шаг ✓ |
| 4.4 | Шаг 2: **Trade URL** → **Сохранить** | Успех; шаг ✓; чеклист готовности: «Ссылка на обмен указана» |
| 4.5 | (Опционально) Шаг 3: **Расширение** | Подключение на той же странице — для автообмена |
| 4.6 | Onboarding: «Аккаунт готов к торговле» | Ссылки на каталог и инвентарь |
| 4.7 | **Продать** → `/sell/inventory` | Загрузка инвентаря |
| 4.8 | **Обновить из Steam** (при необходимости) | Список предметов; счётчик синхронизации |

Для **покупки** достаточно шагов 4.1–4.4 и баланса на кошельке (если $0 — **Кошелёк** → **Тестовое пополнение**).

### Где взять Trade URL

Steam → Инвентарь → «Обмен» → «Кто может мне отправлять предложения обмена?» → скопировать ссылку.

Формат: `https://steamcommunity.com/tradeoffer/new/?partner=...&token=...`

### Если инвентарь пустой

| Причина | Что сделать |
|---------|-------------|
| Инвентарь Steam закрыт | Steam → Настройки → Конфиденциальность → **Публичный** |
| Вошли не под тем аккаунтом | **Выйти** → войти через Steam снова |
| Только медали/значки | Это нормально — они видны, но кнопка **«Недоступен»** (не tradable) |
| Нет CS2-предметов | Нужны предметы из Counter-Strike 2 |

### Сценарий B. Продавец — выставить лот из Steam (15 мин)

**Предусловия:** выполнен раздел 4 (Steam + Trade URL).

| # | Шаг | Ожидание |
|---|-----|----------|
| B1 | Войти через **Steam** | Каталог |
| B2 | Шапка → **Продать** | Страница инвентаря |
| B3 | Найти **tradable** предмет (кнопка **Выставить** активна) | — |
| B4 | **Выставить** → цена **1000** ($) → Submit | Редирект на **Сделки** → вкладка **Мои лоты** (`/deals?tab=listings`) |
| B5 | Проверить статус лота | **Активен**; комиссия ~5%, выплата ~95% |

### Сценарий C. Покупатель + Admin — полная сделка (20 мин)

**Нужно:** 2 окна с **разными Steam** + Mock Admin в инкогнито.

**Предусловия:**

## 6. Тест с двумя Steam-аккаунтами (рекомендуемая схема)

```
┌─────────────────────┐     ┌─────────────────────┐     ┌─────────────────────┐
│  Chrome (обычный)   │     │  Chrome (инкогнито) │     │  Firefox / 2-е инк. │
│  Steam: АККАУНТ A   │     │  Steam: АККАУНТ B   │     │  Mock: ADMIN        │
│  → Продать, Сделки  │     │  → Каталог → Купить │     │  → Complete trade   │
└─────────────────────┘     └─────────────────────┘     └─────────────────────┘
```

Один Steam-аккаунт может **сам** выставить лот и **в другом окне** (другой Steam) купить его. Mock Buyer/Seller для ручного теста не нужны.

**Не смешивайте** в одной вкладке: Steam login → Mock → Link Steam — Steam «переедет» и сессии запутаются.

## 7. Статусы (шпаргалка)

### Заказ (Order)

| Статус | Значение |
|--------|----------|
| `WAITING_TRADE` | Ожидание обмена; можно отменить (покупатель) |
| `COMPLETED` | Сделка успешна |
| `CANCELED` | Отмена покупателем |
| `DISPUTE` | Спор; ждёт решения Admin |
| `FAILED` | Не состоялась |

### Лот (Lot)

| Статус | Значение |
|--------|----------|
| `ACTIVE` | В каталоге |
| `RESERVED` | В процессе покупки |
| `SOLD` | Продан |
| `BLOCKED` | Заблокирован Admin |

## 8. Что НЕ тестируем на этом staging

- Реальные USDT-депозиты и выводы
- Реальный trade offer в Steam (отправка/приём в клиенте Steam)
- Автоматическое завершение сделки без Admin (mock trade)
- Тестовое пополнение доступно всем залогиненным пользователям на staging (не только Admin)
