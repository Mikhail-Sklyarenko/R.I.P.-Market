# Инструкция для тестировщика — staging p2pcs.ru

Пошаговое руководство для ручного тестирования **R.I.P. Market** на закрытом staging.

**URL:** https://p2pcs.ru  
**Дата актуализации:** июль 2026

---

## 1. Что тестируем на этом staging

| Включено | Описание |
|----------|----------|
| ✅ Вход через **Steam** | Реальная авторизация OpenID |
| ✅ **Реальный инвентарь** CS2 | Предметы подтягиваются из Steam-аккаунта |
| ✅ Выставление лотов | Продажа скинов на маркетплейсе |
| ✅ Покупка / отмена / споры | С mock-завершением сделок |
| ✅ Mock **Admin** | Завершение сделок, модерация |
| ❌ USDT / crypto-кошелёк | Не подключён |
| ❌ Реальный обмен в Steam | Сделки завершаются через mock (Admin) |

---

## 2. Перед началом

### 2.1. Требования

- Браузер: **Chrome** или **Firefox** (последняя версия)
- Аккаунт **Steam** с игрой CS2
- Инвентарь CS2 в Steam: **публичный** (не «только для друзей»)
- Желательно **tradable** скины (медали и значки видны, но их нельзя выставить)

### 2.2. Как открыть сайт

| Правильно | Неправильно |
|-----------|-------------|
| https://p2pcs.ru/login | api.p2pcs.ru |
| https://www.p2pcs.ru/login | http без редиректа на старый кэш |

Если сайт не открывается:

1. **Ctrl+Shift+R** (Mac: **Cmd+Shift+R**) — жёсткое обновление
2. Режим **инкогнито**
3. Отключить VPN
4. Написать разработчику: скрин + F12 → Console

### 2.3. Важно про аккаунты

**Один Steam = один пользователь на платформе.**

- Нельзя привязать два разных Steam к одному аккаунту
- Нельзя смотреть инвентарь чужого Steam, будучи залогиненным под другим пользователем
- **Mock Buyer / Seller / Admin** — общие тестовые аккаунты для всех (не личные)

**Типичная ошибка:** вошли через Steam, потом привязали Steam к Mock Seller — Steam «переехал» на другого пользователя. Старая сессия в браузере показывает пустой инвентарь.

**Решение:** **Выйти** → войти снова через **Steam** (или нужную mock-роль).

## 3. Роли и способы входа

Страница входа: https://p2pcs.ru/login

### 3.1. Steam (основной способ)

1. Вкладка **Steam**
2. **Sign in with Steam**
3. Подтвердить вход в Steam
4. После входа вы попадёте в каталог (роль **Buyer** по умолчанию)

У каждого нового Steam создаётся **отдельный** пользователь со **своим** инвентарём.

### 3.2. Mock (вспомогательный, для Admin)

1. Вкладка **Mock**
2. Выбрать роль: **Buyer** / **Seller** / **Admin**
3. **Continue as mock …**

| Роль | Зачем нужна на staging |
|------|------------------------|
| **Admin** | Завершить сделку (mock trade), споры, модерация |
| **Seller** | Тест без своего Steam (mock-инвентарь **не** используется — инвентарь реальный только с привязанным Steam) |
| **Buyer** | Тест покупки без Steam (без привязанного Steam инвентаря нет) |

**Рекомендация:** продавец и покупатель — через **разные Steam** в **разных окнах браузера**. Admin — через Mock в отдельном инкогнито.

## 4. Подготовка аккаунта продавца

Выполните **один раз** перед сценарием продажи.

| Шаг | Действие | Ожидание |
|-----|----------|----------|
| 4.1 | Войти через **Steam** | Успешный вход, отображается ник |
| 4.2 | Открыть **Аккаунт** (меню профиля) | Страница `/account` |
| 4.3 | Проверить **Steam ID** | Отображается ваш SteamID64 |
| 4.4 | Вставить **Trade URL** → **Сохранить** | Сообщение об успехе; чеклист: «Ссылка на обмен указана» |
| 4.5 | Открыть **Продать** → `/sell/inventory` | Загрузка инвентаря |
| 4.6 | Нажать **Обновить из Steam** (при необходимости) | Список предметов; счётчик синхронизации |

### Где взять Trade URL

Steam → Инвентарь → «Обмен» → «Кто может мне отправлять предложения обмена?» → скопировать ссылку.

Формат: `https://steamcommunity.com/tradeoffer/new/?partner=...&token=...`

### Если инвентарь пустой

| Причина | Что сделать |
|---------|-------------|
| Инвентарь Steam закрыт | Steam → Настройки → Конфиденциальность → **Публичный** |
| Вошли не под тем аккаунтом | **Выйти** → войти через Steam снова |
| Только медали/значки | Это нормально — они видны, но кнопка **«Недоступен»** (не tradable) |
| Нет CS2-предметов | Нужны предметы из Counter-Strike 2 |

## 5. Пользовательские сценарии

### Сценарий A. Гость — каталог без входа (5 мин)

**Цель:** сайт открывается, каталог доступен.

| # | Шаг | Ожидание |
|---|-----|----------|
| A1 | Открыть https://p2pcs.ru/catalog | Список лотов (может быть пустым) |
| A2 | Нажать **Открыть** на любом лоте | Карточка: название, цена, износ |
| A3 | Нажать **Купить** / **Войти для покупки** | Редирект на логин или checkout |

---

### Сценарий B. Продавец — выставить лот из Steam (15 мин)

**Предусловия:** выполнен раздел 4 (Steam + Trade URL).

| # | Шаг | Ожидание |
|---|-----|----------|
| B1 | Войти через **Steam** | Каталог или главная |
| B2 | Шапка → **Продать** | Страница инвентаря |
| B3 | Найти **tradable** предмет (кнопка **Выставить** активна) | — |
| B4 | **Выставить** → цена **1000** ($) → Submit | Редирект на **Мои лоты** |
| B5 | Проверить статус лота | **Активен**; комиссия ~5%, выплата ~95% |
| B6 | Открыть каталог в **инкогнито** | Лот виден в списке |

**Баг, если:** лот не появился в каталоге; ошибка при сохранении цены.

---

### Сценарий C. Покупатель + Admin — полная сделка (20 мин)

**Нужно:** 2 окна браузера (обычное + инкогнито), 2 Steam-аккаунта **или** Steam покупатель + mock Admin.

**Предусловия:**

- Лот из сценария B выставлен
- У покупателя есть баланс на кошельке (если $0 — попросите разработчика пополнить mock-депозитом или используйте Admin → Кошелёк → Mock deposit на **своём** admin-аккаунте не поможет покупателю)

#### Окно 1 — Покупатель (Steam)

| # | Шаг | Ожидание |
|---|-----|----------|
| C1 | Войти через **другой Steam** | Каталог |
| C2 | Проверить баланс в шапке | Сумма ≥ цены лота |
| C3 | Открыть лот → **Купить сейчас** | Checkout |
| C4 | **Подтвердить покупку** | Страница сделки `/orders/:id` |
| C5 | Статус сделки | **WAITING_TRADE** |
| C6 | **Кошелёк** | Сумма в **Hold** (заморожена) |

#### Окно 2 — Admin (Mock)

| # | Шаг | Ожидание |
|---|-----|----------|
| C7 | Войти **Mock → Admin** | `/admin/orders` |
| C8 | Открыть сделку из C4 | Карточка заказа |
| C9 | **Complete trade (mock)** | Статус → **COMPLETED** |

#### Проверка

| # | Шаг | Ожидание |
|---|-----|----------|
| C10 | Покупатель: уведомления (колокольчик) | Событие о завершении |
| C11 | Продавец (окно 1, другой Steam): **Мои сделки** | Сделка завершена; выплата ~95% |

## 6. Тест с двумя Steam-аккаунтами (рекомендуемая схема)

```
┌─────────────────────┐     ┌─────────────────────┐     ┌─────────────────────┐
│  Chrome (обычный)   │     │  Chrome (инкогнито) │     │  Firefox / 2-е инк. │
│  Steam: ПРОДАВЕЦ    │     │  Steam: ПОКУПАТЕЛЬ  │     │  Mock: ADMIN        │
│  → Продать           │     │  → Каталог → Купить │     │  → Complete trade   │
└─────────────────────┘     └─────────────────────┘     └─────────────────────┘
```

**Не смешивайте** в одной вкладке: Steam login → Mock Seller → Link Steam — Steam «переедет» и сессии запутаются.

## 7. Статусы (шпаргалка)

### Заказ (Order)

| Статус | Значение |
|--------|----------|
| `WAITING_TRADE` | Ожидание обмена; можно отменить (покупатель) |
| `COMPLETED` | Сделка успешна |
| `CANCELED` | Отмена покупателем |
| `DISPUTE` | Спор; ждёт решения Admin |
| `FAILED` | Не состоялась |

### Лот (Lot)

| Статус | Значение |
|--------|----------|
| `ACTIVE` | В каталоге |
| `RESERVED` | В процессе покупки |
| `SOLD` | Продан |
| `BLOCKED` | Заблокирован Admin |

---

## 8. Что НЕ тестируем на этом staging

- Реальные USDT-депозиты и выводы
- Реальный trade offer в Steam (отправка/приём в клиенте Steam)
- Автоматическое завершение сделки без Admin (mock trade)
- Mock deposit у Buyer/Seller в UI (форма видна только **Admin**; для пополнения покупателя — через разработчика)

## 9. Как сообщить о баге

Для каждого бага укажите:

1. **Сценарий и шаг** (например: «C4 — подтверждение покупки»)
2. **Способ входа** (Steam / Mock Buyer / Admin)
3. **URL** из адресной строки
4. **Ожидание** и **факт**
5. **Скриншот** экрана
6. **Консоль браузера:** F12 → Console → скрин красных ошибок
7. **Браузер и ОС** (Chrome 131, Windows 11)
