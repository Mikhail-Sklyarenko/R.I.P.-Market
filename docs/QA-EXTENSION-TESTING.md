# Инструкция для тестировщика — Chrome-расширение R.I.P Market

Подробное руководство по ручному тестированию **автоотправки trade offer** на staging **p2pcs.ru**.

**Сайт:** https://p2pcs.ru  
**Версия расширения:** [browser-extension-v0.5.0](https://github.com/Mikhail-Sklyarenko/R.I.P.-Market/releases/tag/browser-extension-v0.5.0)  
**Дата:** июль 2026

Общие сценарии staging (каталог, кошелёк, споры): [QA-STAGING-p2pcs.md](./QA-STAGING-p2pcs.md).

---

## 1. Что мы тестируем

Расширение для **продавца**: когда покупатель оплатил лот, расширение в Chrome:

1. получает задачу с сервера;
2. открывает страницу обмена Steam у покупателя;
3. добавляет нужный скин в offer и отправляет обмен;
4. сообщает сайту о прогрессе (фазы на странице сделки).

**На staging сейчас:**

| Включено | Не включено |
|----------|-------------|
| Установка и подключение расширения | Реальное завершение сделки в Steam (без Admin) |
| Автоотправка trade offer | USDT / crypto |
| Прогресс фаз на странице заказа | Полный settlement с реальными деньгами |
| Mock Admin для завершения сделки | Firefox (только Chrome) |

## 2. Что понадобится

### 2.1. Оборудование и софт

- **Google Chrome** (последняя версия). Firefox и Safari **не подходят**.
- **Два Steam-аккаунта** с CS2 (продавец и покупатель).
- **Steam Mobile** на телефоне продавца (для подтверждения Steam Guard, если попросит).
- Стабильный интернет, без агрессивного VPN (Steam может капризничать).

### 2.2. Аккаунты

| Роль | Где входить | Что настроить |
|------|-------------|---------------|
| **Продавец** | Chrome (обычное окно) + p2pcs.ru + steamcommunity.com | Trade URL, tradable скин, **расширение** |
| **Покупатель** | Chrome (инкогнито) или другой браузер | Trade URL, **баланс на кошельке** ≥ цены лота |
| **Admin** | Mock Admin (инкогнито) | Завершение сделки после отправки offer |

**Важно:** продавец работает **в одном профиле Chrome**, где одновременно открыты p2pcs.ru и steamcommunity.com под **одним и тем же** Steam.

### 2.3. Схема окон

```
┌──────────────────────────────┐   ┌──────────────────────────────┐   ┌──────────────────────────────┐
│ Chrome (обычный)             │   │ Chrome (инкогнито)           │   │ Любой браузер (инкогнито)    │
│ Steam: ПРОДАВЕЦ              │   │ Steam: ПОКУПАТЕЛЬ            │   │ Mock: ADMIN                  │
│ + расширение R.I.P Market    │   │ p2pcs.ru → Купить            │   │ Complete trade (mock)        │
└──────────────────────────────┘   └──────────────────────────────┘   └──────────────────────────────┘
```

## 3. Установка расширения (один раз)

### Шаг 1 — скачать

1. Откройте [Releases на GitHub](https://github.com/Mikhail-Sklyarenko/R.I.P.-Market/releases).
2. Найдите релиз **`browser-extension-v0.5.0`** (или новее).
3. Скачайте файл **`rip-market-browser-extension-v0.5.0.zip`**.

### Шаг 2 — распаковать

Распакуйте zip в любую папку. Внутри должна быть папка **`dist`** с файлом `manifest.json`.

Пример структуры:

```
Загрузки/rip-market/
  └── dist/
        ├── manifest.json
        ├── background/
        ├── popup/
        └── ...
```

### Шаг 3 — установить в Chrome

1. Откройте `chrome://extensions`
2. Включите **«Режим разработчика»** (переключатель справа вверху)
3. Нажмите **«Загрузить распакованное расширение»**
4. Выберите папку **`dist`** (ту, где лежит `manifest.json`)
5. Убедитесь, что расширение **«R.I.P Market Trade Assistant»** появилось в списке и **включено**

### Шаг 4 — проверить ID расширения

На карточке расширения в Chrome указан **ID**. Для staging он должен быть:

```
gmmlnkjdbcoojbhndjcfehojknjamaoj
```

Если ID **другой** — вы скачали не тот zip или старую версию. Скачайте актуальный релиз с GitHub.

### Шаг 5 — закрепить (по желанию)

Нажмите на иконку «пазл» в панели Chrome → закрепите **R.I.P Market**, чтобы видеть статус.

## 4. Подготовка аккаунта продавца

Выполните **до** теста расширения.

| # | Действие | Ожидание |
|---|----------|----------|
| 4.1 | Откройте https://p2pcs.ru/login → вкладка **Steam** → войдите | Успешный вход, виден ник |
| 4.2 | В **том же Chrome** откройте https://steamcommunity.com | Вы залогинены под **тем же** Steam, что на p2pcs.ru |
| 4.3 | p2pcs.ru → **Аккаунт** (`/account`) | Отображается Steam ID |
| 4.4 | Вставьте **Trade URL** → **Сохранить** | Сообщение об успехе |
| 4.5 | **Продать** → `/sell/inventory` → **Обновить из Steam** | Видны tradable скины |
| 4.6 | Выставьте лот (цена, например **$10**) | Лот **Активен** в «Мои лоты» |
| 4.7 | **Аккаунт** → блок **«Расширение для автообмена»** → **Подключить расширение** | Текст **«Расширение подключено…»**, статус **«Подключено»** |

### Если кнопки «Подключить расширение» нет

Вместо неё серый блок «Установите расширение…» — Chrome не видит расширение. Проверьте шаг 3 (установка, ID, расширение включено). Обновите страницу **Ctrl+Shift+R**.

### Если ошибка при подключении

| Текст ошибки | Что сделать |
|--------------|-------------|
| «Установите расширение…» | Переустановите из zip, проверьте ID |
| «Could not establish connection» | Обновите страницу; перезагрузите расширение на `chrome://extensions` |
| Другая ошибка | Скрин + F12 → Console → отправьте разработчику |

## 6. Основной сценарий — happy path (30–40 мин)

**Цель:** расширение само отправляет trade offer после покупки.

### Часть A — покупка

| # | Кто | Шаг | Ожидание |
|---|-----|-----|----------|
| 6.1 | Покупатель | Каталог → открыть лот продавца → **Купить сейчас** → подтвердить | Редирект на `/orders/:id` |
| 6.2 | Покупатель | Статус сделки | **WAITING_TRADE** |
| 6.3 | Покупатель | **Кошелёк** | Сумма в **Hold** (заморожена) |

### Часть B — работа расширения (продавец)

| # | Кто | Шаг | Ожидание |
|---|-----|-----|----------|
| 6.4 | Продавец | Открыть **Мои сделки** → эту сделку | Страница заказа |
| 6.5 | Продавец | Блок **«Статус расширения»** | Виден блок с фазой (не «Задача ещё не создана») |
| 6.6 | Продавец | Подождать 15–30 сек | Фазы меняются (см. таблицу ниже) |
| 6.7 | Продавец | Может открыться вкладка Steam | Страница trade offer покупателя |
| 6.8 | Продавец | Если Steam Guard на телефоне | Подтвердить обмен в **Steam Mobile** |
| 6.9 | Продавец | Финальная фаза | **«Обмен отправлен»** / фаза `OFFER_SENT` |

### Ожидаемые фазы (по порядку)

| Фаза (техн.) | Текст на сайте | Что происходит |
|--------------|----------------|----------------|
| `ACKED` | Задача принята расширением | Расширение забрало задачу |
| `TRADE_PAGE_OPENED` | Открыта страница обмена в Steam | Вкладка Steam |
| `OFFER_DRAFTED` | Обмен подготовлен | Форма обмена готова |
| `ITEM_SELECTED` | Предмет добавлен в обмен | Нужный скин выбран |
| `OFFER_SUBMITTED` | Обмен отправляется | Отправка в Steam |
| `CONFIRM_PENDING` | Подтвердите в Steam Guard | Нужно подтверждение на телефоне |
| `OFFER_SENT` | Обмен отправлен | Успех |

Не все фазы могут мелькнуть быстро — это нормально. Главное — дойти до **OFFER_SENT** без ошибки.

### Часть C — завершение на staging

| # | Кто | Шаг | Ожидание |
|---|-----|-----|----------|
| 6.10 | Admin | https://p2pcs.ru/login → **Mock → Admin** | Вход в админку |
| 6.11 | Admin | `/admin/orders` → открыть сделку | Карточка заказа |
| 6.12 | Admin | **Complete trade (mock)** | Статус **COMPLETED** |
| 6.13 | Покупатель / продавец | Обновить страницу сделки | **COMPLETED**, уведомление |

## 7. Дополнительные сценарии

### 7.1. Кнопка «Повторить сейчас»

На странице сделки (продавец), пока обмен не завершён, есть кнопка **«Повторить сейчас»**. Нажмите — расширение должно снова опросить сервер (фаза может обновиться в течение ~15 с).

**Ожидание:** без зависания, без дублирования offer (если offer уже отправлен — фаза остаётся `OFFER_SENT`).

### 7.2. Переподключение расширения

1. **Аккаунт** → **Отключить** → **Подключить расширение** снова  
2. Повторите покупку (новый лот или отмените старую сделку)

**Ожидание:** подключение успешно, новая сделка обрабатывается.

### 7.3. Другой Steam в Chrome (негативный)

1. В Chrome залогиньтесь в steamcommunity.com под **чужим** Steam (не продавцом)  
2. Попробуйте сделку

**Ожидание:** ошибка вроде «В Chrome залогинен другой Steam-аккаунт» / код `STEAM_ACCOUNT_MISMATCH`. Ручной fallback на странице сделки остаётся доступен.

### 7.4. Покупатель без Trade URL (негативный)

Уберите Trade URL у покупателя → купите лот.

**Ожидание:** расширение не может отправить offer; ошибка про Trade URL покупателя.

### 7.5. Ручной fallback

Если автоотправка не сработала, на странице сделки продавца есть блок ручного ввода trade reference / offer. Используйте его как запасной путь и **зафиксируйте баг** с описанием, почему авто не сработало.

## 8. Типичные проблемы

| Симптом | Вероятная причина | Что сделать |
|---------|-------------------|-------------|
| «Задача расширения ещё не создана» | Покупка только что прошла | Подождать 5–10 с, обновить страницу |
| Фаза не двигается > 2 мин | Steam не залогинен / другой аккаунт | Проверить steamcommunity.com под продавцом |
| `INVENTORY_NOT_LOADED` | Steam rate limit | Открыть steamcommunity.com, обновить сделку; в popup расширения можно указать Steam Web API key |
| `ITEM_MISSING` | Скин продан или не в инвентаре | **Обновить из Steam** на сайте, пересоздать лот |
| `TRADE_HOLD_BLOCKED` | Steam trade hold | Подождать или отправить offer вручную |
| `CONFIRM_PENDING` долго | Guard не подтверждён | Steam Mobile → подтвердить обмен |
| `SESSION_REVOKED` | Сессия расширения истекла | **Аккаунт** → подключить снова |
| Сайт не видит расширение | Не тот ID / не Chrome | Проверить ID, переустановить из zip |
| Вкладка Steam не открывается | Блокировщик popup | Разрешить popup для steamcommunity.com |

## 9. Что писать в баг-репорте

При любой ошибке приложите:

1. **Скриншот** страницы сделки (блок «Статус расширения»)
2. **URL** сделки (`/orders/...`)
3. **Роль** (продавец / покупатель)
4. **Время** (МСК)
5. **Фаза** и текст ошибки (если есть)
6. **Скрин** `chrome://extensions` с ID расширения
7. **F12 → Console** на p2pcs.ru (если есть красные ошибки)

Шаблон:

```
Заголовок: [Extension] Кратко что сломалось

Шаги:
1. ...
2. ...

Ожидание: ...
Факт: ...
Фаза: OFFER_SUBMITTED / ошибка: ...
Сделка: https://p2pcs.ru/orders/...
Steam продавца: 76561198... (последние 4 цифры)
Браузер: Chrome 1xx, Windows/macOS
```

## 12. Вопросы разработчику

Пишите, если:

- не можете пополнить баланс покупателю;
- расширение не подключается при правильном ID;
